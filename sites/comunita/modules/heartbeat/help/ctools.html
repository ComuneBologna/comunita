As of drupal7, Heartbeat extends fully on the Chaos tools suite. It does not have any functions by itself anymore to retrieve or perform other CRUD actions to the database. We need to use the ctools functions instead. Available functions are listed beneath.

Exportables

Data types managed by ctools extended on the export and export_ui plugin is a very conventional and maintainable approach to handle object storage in heartbeat. Underneath is a list of functions to use if you want to load or save data to the heartbeat storage types.

  • ctools_export_crud_load
  • ctools_export_crud_load_all
  • ctools_export_crud_save
  • ctools_export_crud_delete
  • ctools_export_crud_import
  • ctools_export_crud_export

See ctools documentation or advanced help to read more about exportables.

Heartbeat template object

The templates can be found at "admin/structure/heartbeat_ui". You can edit, delete, revert, disable this type.

Heartbeat stream configiguration object

The streams can be found at "admin/structure/heartbeat_streams". You can edit, delete, revert, disable this type.
